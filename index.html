    siddhi-io-tcp

    The siddhi-io-tcp extension is an extension to Siddhi that allows to receive and publish events through TCP.

    Find some useful links below:

    • Source code
    • Releases
    • Issue tracker

    Latest API Docs

    Latest API Docs is 2.0.14.

    How to use

    Using the extension in WSO2 Stream Processor

    • You can use this extension in the latest WSO2 Stream Processor that is a part of WSO2 Analytics offering, with editor, debugger and simulation support.

    • This extension is shipped by default with WSO2 Stream Processor, if you wish to use an alternative version of this extension you can replace the component jar that can be found in the <STREAM_PROCESSOR_HOME>/lib directory.

    Using the extension as a java library

    • This extension can be added as a maven dependency along with other Siddhi dependencies to your project.
         <dependency>
        <groupId>org.wso2.extension.siddhi.io.tcp</groupId>
        <artifactId>siddhi-io-tcp</artifactId>
        <version>x.x.x</version>
     </dependency>

    Features

    • tcp ((Sink))

      A Siddhi application can be configured to publish events via the TCP transport by adding the @Sink(type = 'tcp') annotation at the top of an event stream definition.

    • tcp ((Source))

      A Siddhi application can be configured to receive events via the TCP transport by adding the @Source(type = 'tcp') annotation at the top of an event stream definition.

      When this is defined the associated stream will receive events from the TCP transport on the host and port defined in the system.
